🧠 PENERAPAN PILAR OOP

1. ENKAPSULASI

Status internal film/kursi disembunyikan dan hanya dapat diubah secara aman lewat metode pesanTiket() dan batalTiket(). Jumlah tiket per pembeli juga dijaga oleh batas aturan (maks 5 tiket).

2. PEWARISAN (INHERITANCE)

Kelas Film dan Studio mewarisi data dasar dari kelas induk. Ini menghindari redundansi properti dasar seperti nama, harga, dan ketersediaan.

3. POLIMORFISME

Metode tampilkanInfo() di kelas induk di‑override pada kelas Film dan Studio untuk menampilkan properti spesifik masing‑masing.

4. INTERAKSI OBJEK

Objek Pembeli memanggil metode pada objek Film dan Kursi untuk mengubah ketersediaan. Hubungan pemesanan diabadikan menggunakan objek Tiket.

        function pesanTiket(){
            if(!selectedFilm){
                alertMessage("Pilih film terlebih dahulu.", "error");
                return;
            }
            if(selectedSeats.length===0){
                alertMessage("Pilih setidaknya satu kursi.", "error");
                return;
            }
            // tandai kursi sebagai dipesan
            selectedSeats.forEach(kode=>{
                const seat=kursiData.find(s=>s.kode===kode);
                if(seat) seat.dipesan=true;
            });
            // buat data tiket (sederhana)
            const tiket={
                pembeli:selectedPembeli,
                film:selectedFilm.judul,
                studio:studios[selectedStudio].nama,
                time:selectedTime,
                seats:[...selectedSeats],
                total:document.getElementById('summaryTotal').textContent
            };

            renderSeats();
            renderTiketList();
            renderLaporan();
            alertMessage(`Berhasil memesan ${selectedSeats.length} tiket untuk ${selectedPembeli}.`, "success");
